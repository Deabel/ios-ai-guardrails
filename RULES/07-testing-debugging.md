# Testing and Debugging

## 测试

### 默认要求

非平凡新逻辑应补测试，或者说明为什么暂不补。

### 测试重点

至少覆盖：

- happy path
- failure path
- edge case

### 推荐

- Swift Testing 或 XCTest，遵循项目现状
- 可注入依赖
- 稳定、可重复、低耦合

### 避免

- 对实现细节过度断言
- 睡眠等待式异步测试
- 随机性测试

## 调试

排查问题时优先输出：

1. 复现路径
2. 可能根因
3. 最小修复方案
4. 回归风险点
5. 建议测试点

## 日志

日志应包含上下文：

- 输入条件
- 关键状态
- 错误信息
- 资源标识

避免满屏 `print("here")` 式原始巫术。
