# Swift Style

## 总体风格

代码应该让团队成员一眼能读懂。

### 推荐

- 小类型
- 小函数
- 早返回
- 意图明确的命名
- 默认不可变
- 明确访问控制

### 避免

- 过深嵌套
- 超长函数
- 一个 extension 塞满无关职责
- 为炫技而写的泛型或协议体操
- 没有注释说明前提的强制解包

## 命名

- 类型：`PascalCase`
- 变量 / 方法 / 属性：`camelCase`
- 布尔值：`is...` / `has...` / `can...` / `should...`

## 可选值处理

优先：

- `guard let`
- `if let`
- 明确的默认值
- 明确错误抛出

避免：

- 到处 `!`
- 静默 `??` 掩盖真正错误

## 错误处理

优先：

- `throws`
- `Result`
- 携带上下文的错误日志

避免：

- 空 `catch`
- 随手 `try?`
- 所有错误都变成 `nil`

## 注释

注释要解释：

- 为什么这样做
- 有什么边界条件
- 有什么约束或坑

不要注释显而易见的代码。
